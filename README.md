## ✨ Features

- **24+ Tools Database**: Comprehensive collection of development and business tools
- **8 Categories**: From Version Control to Productivity Tools
- **Advanced Filtering**: Search, category, segment, type, and pricing filters
- **Modern UI**: Beautiful responsive design with Tailwind CSS
- **Real-time Search**: Instant tool discovery
- **Detailed Tool Cards**: Pricing, pros/cons, API availability, and more

## 📊 Categories

### 🖥️ Segment 1: Development & Data Ecosystem
- **Version Control & Collaboration** (GitHub, GitLab, Bitbucket)
- **Programming IDEs & Editors** (VS Code, IntelliJ, Cursor, PyCharm, WebStorm, Sublime, Vim)
- **Cloud Platforms** (Vercel, Railway, Heroku)
- **DevOps & CI/CD** (Docker, Kubernetes)
- **Databases** (PostgreSQL, MongoDB, Redis)
- **AI Development Tools** (ChatGPT, Claude)

### 🎯 Segment 2: Business & AI Tools
- **Design & Creative Tools** (Figma, Canva)
- **Productivity Tools** (Notion, Slack)

## 📦 Installation

```bash
# Clone repository
git clone https://github.com/rajeevrajora77-lab/tool-app.git
cd tool-app

# Install dependencies
npm install

# Run development server
npm run dev

# Build for production
npm run build
```

## 🔍 Usage

1. **Search**: Type in the search bar to find tools by name, description, or category
2. **Filter by Segment**: Choose between Development & Data or Business & Tools
3. **Filter by Category**: Select specific tool categories
4. **Filter by Type**: Open-source, Proprietary, or Open Core
5. **Free Tier Toggle**: Show only tools with free tiers
6. **Click Cards**: Visit official websites and explore tool details

## 📋 Data Structure

Each tool includes:
- Name and official website link
- Category and segment classification
- Founded year and type (open-source/proprietary)
- Detailed description
- Pricing tiers with features
- Pros and cons analysis
- API availability status
- Tags and additional metadata

## 🚀 Deployment

The app is configured for instant deployment on:

- **Railway**: Deploy with `railway up`

- **GitHub Pages**: Deploy with GitHub Actions
